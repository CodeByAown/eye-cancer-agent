# Technology Review — Choices, Rationale, Alternatives

Every core technology reviewed: why chosen, what was considered, verdict. Optimized for a demoable, maintainable, scalable medical platform that reuses mature open-source components.

| Tech                                                          | Why chosen                                                                                                               | Alternatives considered                                                                        | Verdict                                         |
| ------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **Next.js 15**                                                | SSR/RSC/streaming, App Router, best-in-class React meta-framework, Vercel deploy, marketing + app in one                 | Remix (smaller ecosystem), plain Vite SPA (no SSR/SEO), SvelteKit (smaller talent pool)        | **Keep**                                        |
| **React**                                                     | Largest ecosystem, MediaPipe/Recharts/shadcn all React-first, hiring pool                                                | Vue, Svelte, Solid                                                                             | **Keep**                                        |
| **TypeScript**                                                | Type safety across a large medical app; shared types from OpenAPI                                                        | plain JS (unsafe at this scale)                                                                | **Keep**                                        |
| **Tailwind CSS**                                              | Fast, consistent, token-driven theming for glass/dark-light                                                              | CSS Modules, styled-components (runtime cost), vanilla-extract                                 | **Keep** (+ shadcn/ui)                          |
| **shadcn/ui** _(added)_                                       | Ownable, accessible (Radix) component base; not a locked library                                                         | MUI (heavy, opinionated), Chakra, Mantine                                                      | **Adopt**                                       |
| **Framer Motion**                                             | Premium animations, `prefers-reduced-motion`, React-native fit                                                           | GSAP (imperative), React Spring (lower-level), CSS-only (limited)                              | **Keep**                                        |
| **FastAPI**                                                   | Async, Pydantic v2 validation, auto OpenAPI, Python = ML language, great DX                                              | Flask (less async/validation), Django (heavier), Node/Nest (ML not native)                     | **Keep**                                        |
| **PostgreSQL**                                                | Relational integrity for medical data, JSONB flexibility, **pgvector** for chat RAG, mature                              | MySQL (weaker JSON/vector), MongoDB (weak relational integrity)                                | **Keep**                                        |
| **Redis**                                                     | Queue backend (Arq), caching, rate limiting, pub/sub for SSE                                                             | RabbitMQ/Kafka (heavier for our scale), in-memory (not durable)                                | **Keep**                                        |
| **Docker**                                                    | Reproducible envs, compose for dev, images for prod                                                                      | Podman (fine alt), bare metal (not reproducible)                                               | **Keep**                                        |
| **MediaPipe**                                                 | Only mature in-browser realtime face+iris; privacy (local WASM)                                                          | TensorFlow.js face models (less turnkey), OpenCV.js (no iris/pose), cloud CV (privacy/latency) | **Keep** (core of scanner)                      |
| **OpenCV**                                                    | Server-side preprocessing, quality gating, classical CV                                                                  | Pillow-only (less capable), scikit-image (slower)                                              | **Keep** (+ Pillow)                             |
| **PyTorch**                                                   | Ecosystem for RETFound/MONAI/timm; research→prod                                                                         | TensorFlow (some models exist but PyTorch dominates our chosen models), JAX (research-only)    | **Keep**                                        |
| **Hugging Face**                                              | Model/dataset hub, `timm`, transformers, hosting                                                                         | manual weight mgmt (worse)                                                                     | **Keep**                                        |
| **ONNX Runtime**                                              | Portable, fast CPU/GPU inference; enables CPU-only MVP + in-browser later                                                | TorchScript (less portable), raw PyTorch serving (heavier)                                     | **Keep** (light models)                         |
| **NVIDIA Triton** _(added, later)_                            | Multi-model GPU serving, dynamic batching for scale                                                                      | TorchServe (simpler, less perf), Ray Serve                                                     | **Adopt at scale**                              |
| **MONAI + nnU-Net** _(added)_                                 | Purpose-built medical imaging (3D/DICOM/seg), Apache-2.0                                                                 | custom pipelines (reinventing), plain PyTorch (no medical transforms)                          | **Adopt** for CT/MRI/seg                        |
| **Arq** _(queue, added)_                                      | Lightweight async Redis queue matching FastAPI async                                                                     | Celery (heavier, sync-oriented), RQ, Dramatiq                                                  | **Adopt** (Celery if complexity grows)          |
| **Clerk**                                                     | Fastest polished auth (MFA/orgs/social/prebuilt UI) for demo timeline                                                    | Auth.js (self-host, less polish), Auth0 (costly), Supabase Auth (fine alt)                     | **Keep** (Auth.js if sovereignty required)      |
| **Cloudflare R2 / S3**                                        | S3-compatible, no egress fees, CDN, signed URLs                                                                          | AWS S3 (egress cost), Supabase Storage (fine alt)                                              | **Keep**                                        |
| **Claude API**                                                | Grounded report/education/chat prose; strong safety adherence; `claude-sonnet-5` cost/latency, `claude-opus-4-8` premium | OpenAI/Gemini (viable), local LLM (weaker, ops burden)                                         | **Keep**                                        |
| **WeasyPrint** _(added)_                                      | HTML/CSS→PDF fidelity reusing design tokens                                                                              | ReactPDF (less CSS), Puppeteer (heavy Chromium)                                                | **Adopt**                                       |
| **SQLAlchemy 2 + Alembic**                                    | Async ORM + migrations                                                                                                   | Tortoise, raw SQL (no migrations)                                                              | **Adopt**                                       |
| **Sentry + Prometheus + Grafana + Evidently + OpenTelemetry** | Errors + infra + ML drift + tracing (see [`10-aiops-monitoring.md`](10-aiops-monitoring.md))                             | Datadog/New Relic (paid), ELK-only (no metrics)                                                | **Adopt**                                       |
| **GitHub Actions**                                            | CI/CD, container scan, preview deploys                                                                                   | GitLab CI, CircleCI                                                                            | **Keep**                                        |
| **pgvector** _(added)_                                        | Chat RAG embeddings in existing Postgres (no new infra)                                                                  | Pinecone/Weaviate/Qdrant (extra service)                                                       | **Adopt** (dedicated store only at large scale) |

## Net changes from v1

**Added:** shadcn/ui, MONAI+nnU-Net, Arq, Triton (later), WeasyPrint, pgvector, Evidently/OTel, DICOM viewer (OHIF, later). **Confirmed:** the entire v1 stack you listed remains the best choice — no replacements needed. **Decision still open:** Clerk vs Auth.js (recommend Clerk for demo).
